native-qemu configuration

The agent loads config in this order: /media/<boot-device>/config.toml, /config.toml, /etc/native-qemu/config.toml, then /etc/native-qemu/config.toml.example. After changing a file on the appliance filesystem, run lbu commit before rebooting.

Networking

mode = "user" requires no host networking setup. Bridge mode requires an existing Linux bridge (normally br0), not a raw Ethernet interface. Macvtap uses bridge_iface as its physical parent and gives the VM a direct attachment; it is not an SSH management path from the host.

Guest services

[docs_server] binds HTTP and DNS only to its bridge interface. Set dhcp_range only when this appliance should provide DHCP on that isolated network.

[shared_folder] starts virtiofsd for one host directory. For a Linux guest:

mkdir -p /mnt/shared
mount -t virtiofs native-qemu-share /mnt/shared

For SMB, define one or more [[smb_share]] entries. A vm_only share is served only on [smb].vm_iface; a global share is served only on [smb].lan_iface and always requires its password file.

The x86 Windows XP VirtIO test profile is bundled at /etc/native-qemu/examples/winxp-virtio.toml. It requires the guest's viostor driver before it can boot from the configured VirtIO disk.

Rescue

Missing disks, required USB devices, or invalid host-service configuration open the rescue shell. Correct the file, run lbu commit if the change should persist, then type exit to retry the agent.
